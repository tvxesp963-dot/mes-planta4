1
☁️ Configurar Google Firebase
Base de datos en la nube — gratis para tu caso de uso
🌐
Crear proyecto Firebase
1. Ve a console.firebase.google.com
2. Haz clic en "Agregar proyecto"
3. Nombre del proyecto: ims-planta4
4. Desactiva Google Analytics (no es necesario) → Crear proyecto
🔑
Obtener credenciales
1. En tu proyecto → icono ⚙️ → Configuración del proyecto
2. Baja hasta "Tus apps" → clic en el icono </> (Web)
3. Nombre de la app: IMS Web → Registrar app
4. Copia el bloque firebaseConfig = { ... }
5. Pégalo en el archivo firebase-config.js reemplazando los valores de ejemplo
// Ejemplo de cómo debe quedar firebase-config.js: const firebaseConfig = { apiKey: "AIzaSyD-tu-clave-real-aqui", authDomain: "ims-planta4.firebaseapp.com", projectId: "ims-planta4", storageBucket: "ims-planta4.appspot.com", messagingSenderId: "987654321", appId: "1:987654321:web:abc123def456" }; initFirebase(firebaseConfig); // No borres esta línea
👥
Activar Authentication (Login de usuarios)
1. Menú izquierdo → Authentication → Comenzar
2. Pestaña "Sign-in method" → Correo electrónico/Contraseña → Habilitar ✓
3. Pestaña "Usuarios" → Agregar usuario
4. Agrega uno por uno los correos y contraseñas de tu equipo (6-15 personas)

Roles sugeridos:
— Administrador: tú + supervisores
— Operador: personal de línea
Tip: El sistema asigna el rol (Admin/Operador) al momento de iniciar sesión desde el selector en la pantalla de login. Puedes cambiar esto en cualquier momento.
🗄️
Activar Firestore Database
1. Menú izquierdo → Firestore Database → Crear base de datos
2. Selecciona "Modo producción" → Siguiente
3. Ubicación: us-central1 (recomendado para México) → Listo
4. Ve a la pestaña "Reglas" y reemplaza todo con esto:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
Haz clic en Publicar después de pegar las reglas.
✅ Firebase listo. Ahora todos los movimientos se sincronizan automáticamente entre todos los usuarios en tiempo real. Cuando un operador registre una salida en su celular, el dashboard del supervisor se actualiza en segundos.
2
🔗 Conectar con tu MES
Sincronización bidireccional de datos de producción
⚙️
Configurar el endpoint desde la app
1. Abre IMS_Firebase_Planta4.html → inicia sesión como Administrador
2. Ve a Admin ⚙️ → sección "Integración MES"
3. Ingresa la URL de tu API (pregúntale al desarrollador de tu MES cuál es el endpoint)
4. Ingresa el API Key o token si tu MES lo requiere
5. Clic en "Probar conexión" — el log te mostrará si funciona
6. Clic en "Guardar configuración"
📡
¿Qué datos se intercambian con el MES?
IMS → MES (cada movimiento registrado):
— Tipo de movimiento (Entrada/Salida/Traslado)
— Part Number, cantidad, ubicación
— Usuario responsable, fecha/hora, OC de referencia

MES → IMS (al sincronizar):
— Producción del día por máquina (S1, S2, S3, S4, ASSY1, VC)
— Consumo de materiales por línea
— Estatus de órdenes de producción
🗺️
Mapeo de campos (opcional)
Si tu MES usa nombres de campos diferentes, configura el mapeo en formato JSON:
// Ejemplo: tu MES llama "qty_produced" lo que IMS llama "produccion" { "produccion": "qty_produced", "maquina": "machine_id", "turno": "shift_name", "fecha": "production_date" }
⚠️ Necesitas saber de tu MES: la URL del endpoint, el método (GET/POST), el formato de autenticación (Bearer token, API Key, Basic Auth) y los nombres de los campos. Pídele esta información al desarrollador de tu sistema MES actual.
3
🔔 Alertas y notificaciones
Stock mínimo y notificaciones automáticas
⚠️
Configurar stock mínimo por PN
1. En la app → pestaña 🔔 Alertas
2. En la sección "Umbral de stock mínimo":
— Escribe el Part Number (ej: GH69-46036A)
— Escribe la cantidad mínima (ej: 1000)
— Clic en "+ Agregar"
3. El sistema generará una alerta automática cuando el stock caiga por debajo del mínimo
4. La campana 🔔 del topbar mostrará el número de alertas activas
📲
Instalar la app en el celular (sin App Store)
Los operadores pueden instalar el sistema como una app en su celular:

En Android (Chrome):
1. Abre el archivo HTML en Chrome
2. Menú ⋮ → "Agregar a pantalla de inicio"
3. Aparece como app con ícono propio

En iPhone (Safari):
1. Abre en Safari
2. Botón compartir □↑ → "Agregar a pantalla de inicio"
Para que funcione en red local: Sube el archivo HTML a un servidor interno o usa Google Drive → Compartir → "Cualquiera con el enlace" para que todos accedan desde su celular.
4
🏭 Arranque en planta
Checklist para el primer día de operación
✅
Checklist antes de arrancar
□ Firebase configurado y probado
□ Usuarios creados en Firebase Auth (uno por persona)
□ MES endpoint configurado y probado
□ Stock mínimos definidos para los PN críticos
□ QR generados e impresos para cada Part Number
□ QR pegados en las cajas/racks de cada material
□ Operadores capacitados en el flujo: escanear → cantidad → confirmar
□ Archivo compartido con todos los usuarios (Drive / servidor)
📷
Imprimir los QR codes
1. Abre la app → pestaña 📷 QR
2. Busca cada Part Number → genera su QR
3. Clic en "🖨️ Imprimir"
4. Pega el QR en la caja, rack o bolsa de ese material

Recomendación: Usa papel adhesivo o plástico transparente para proteger los QR en planta
